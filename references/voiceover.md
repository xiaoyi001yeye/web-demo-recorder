# Voiceover Reference

Voiceover is timeline-based: each segment starts at an absolute second in the video and contributes either text to synthesize or an existing audio file to mix.

## Generated Narration

Generated narration uses macOS `say`:

```json
{
  "voice": "Tingting",
  "rate": 220,
  "segments": [
    { "start": 0.4, "text": "Admin view: confirm the project." },
    { "start": 4.5, "text": "Switch to the expert account." }
  ]
}
```

Run:

```bash
node /path/to/web-demo-recorder/scripts/add-voiceover.cjs \
  --input /absolute/path/to/demo.mp4 \
  --timeline /absolute/path/to/voiceover.json \
  --output /absolute/path/to/demo-voiceover.mp4
```

## Pre-Rendered Audio

Use `audioFile` when narration has already been generated by another TTS tool:

```json
{
  "segments": [
    { "start": 0.4, "audioFile": "/absolute/path/to/segment-01.wav" },
    { "start": 4.5, "audioFile": "/absolute/path/to/segment-02.wav" }
  ]
}
```

Or put files in an audio directory and name them by segment number:

```text
segment-01.wav
segment-02.mp3
segment-03.aiff
```

Then run with `--audio-dir /absolute/path/to/audio`.

## Quality Gate

The helper reports overlaps and whether the last narration segment ends before the video ends. Treat either failure as a bad guided demo unless the user explicitly accepts the timing.
